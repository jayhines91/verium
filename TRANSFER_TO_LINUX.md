# Transferring Work to Linux Build Machine

## Option 1: Git Push/Pull (Recommended if repo is synced)

### On macOS (current machine):
```bash
# Commit all changes
git add .
git commit -m "Add Linux build scripts and macOS fixes"

# Push to remote
git push origin $(git branch --show-current)
```

### On Linux machine:
```bash
# Clone or pull latest
git clone <your-repo-url> verium
# OR if already cloned:
cd verium
git pull origin $(git branch --show-current)
```

## Option 2: Create Transfer Package

### On macOS:
```bash
# Create a tarball excluding build artifacts
cd /Users/jay/Code\ Workspace/verium-1.3.5.1/testnet
tar --exclude='*.o' --exclude='*.a' --exclude='.deps' --exclude='autom4te.cache' \
    --exclude='config.log' --exclude='config.status' --exclude='Makefile' \
    --exclude='src/qt/*.o' --exclude='src/qt/*.moc' --exclude='src/qt/*.ui.h' \
    -czf verium-transfer.tar.gz verium/

# Transfer via scp (adjust host/path)
scp verium-transfer.tar.gz user@linux-box:/path/to/destination/
```

### On Linux:
```bash
cd /path/to/destination
tar -xzf verium-transfer.tar.gz
cd verium
./autogen.sh  # Regenerate configure scripts
```

## Option 3: rsync (Best for incremental syncs)

### On macOS:
```bash
rsync -avz --exclude='*.o' --exclude='*.a' --exclude='.deps' \
    --exclude='autom4te.cache' --exclude='config.log' \
    --exclude='config.status' --exclude='Makefile' \
    --exclude='src/qt/*.o' --exclude='src/qt/*.moc' \
    /Users/jay/Code\ Workspace/verium-1.3.5.1/testnet/verium/ \
    user@linux-box:/path/to/verium/
```

## Option 4: GitHub/GitLab (If using remote repo)

If you have a GitHub/GitLab repo:
1. Push changes from macOS
2. Pull on Linux machine
3. Continue building

## What to Transfer

### Essential Files:
- ✅ All source code (`src/`)
- ✅ Build scripts (`build-linux-*.sh`)
- ✅ Configuration files (`configure.ac`, `Makefile.am`, etc.)
- ✅ Dependencies system (`depends/`)
- ✅ Documentation (`doc/`, `release-binaries/linux/`)
- ✅ Testnet configs (`testnet/`)

### Can Skip (will regenerate):
- ❌ `config.log`, `config.status`, `Makefile` (regenerated by configure)
- ❌ Object files (`*.o`)
- ❌ Libraries (`*.a`, `*.so`)
- ❌ Build artifacts (`autom4te.cache`, `.deps/`)

## After Transfer on Linux

1. **Install prerequisites:**
   ```bash
   # For x86_64
   sudo apt-get update
   sudo apt-get install -y build-essential automake libtool pkg-config python3 curl
   
   # For ARM64 (if building ARM)
   sudo apt-get install -y g++-aarch64-linux-gnu binutils-aarch64-linux-gnu
   ```

2. **Regenerate build system:**
   ```bash
   ./autogen.sh
   ```

3. **Build:**
   ```bash
   ./build-linux-x86.sh  # or ./build-linux-arm.sh
   ```

## Quick Check Script

Run this on Linux to verify everything transferred:
```bash
#!/bin/bash
echo "Checking essential files..."
[ -f "build-linux-x86.sh" ] && echo "✅ x86 build script" || echo "❌ Missing x86 script"
[ -f "build-linux-arm.sh" ] && echo "✅ ARM build script" || echo "❌ Missing ARM script"
[ -d "src/qt" ] && echo "✅ Qt source" || echo "❌ Missing Qt source"
[ -d "depends" ] && echo "✅ Depends system" || echo "❌ Missing depends"
[ -f "configure.ac" ] && echo "✅ Configure script" || echo "❌ Missing configure.ac"
echo "Done!"
```
